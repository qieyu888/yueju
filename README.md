觅伴 - 场景化轻社交 APP 产品说明文档1. 产品概述产品名称：觅伴 (暂定名)产品定位：一款主打“反内耗、去聊天化、场景驱动”的线下约玩与轻社交 APP。目标用户：一二线城市中，生活节奏快、渴望线下真实社交但厌倦了无效线上闲聊的年轻群体（18-35岁）。2. 核心产品理念No-Chat (去聊天化)：砍掉繁重的内置 IM（即时通讯）系统。用户间的连接依靠“活动匹配”和“系统通知”完成，最大程度降低社交破冰的心理门槛。场景驱动：以具体的活动（如滑雪、剧本杀、看展、约饭）为核心锚点，解决“不知道聊什么”的尴尬，让社交回归“一起做某事”的本质。高效触达：线下集合前的紧急沟通，直接调用系统原生的电话和短信功能，回归最纯粹的联络方式。3. 核心功能模块详细设计3.1 首页：发现聚会 (Activities)设计风格：沉浸式大图画廊（Carousel），支持横向丝滑滑动（类探探/小红书画廊），具有强烈的视觉冲击力。功能描述：分类筛选：顶部提供快速标签切换（如：全部、滑雪、约饭、运动、看展等）。沉浸式卡片：每张卡片代表一个正在招募的活动。图片/兜底背景：发布者可上传图片；若无图片，系统自动匹配与活动类型相关的高级渐变色（Fallback Background）。关键信息提取：卡片上直观显示发起人、活动类型、时间、地点。参与状态：底部直观显示“已加入 (N/M)”，并展示已加入成员的圆角头像（最多展示3个，超出的显示 +N）。一键报名：用户点击“我要去”即可报名，名额满员时按钮自动置灰禁用。3.2 发起聚会 (Publish)设计风格：底部导航栏中央醒目的凸起“+”号按钮。功能描述：用户随时可以成为发起人（局长）。极简发布流程：只需输入【标题、类型、时间、地点、人数上限、配图（选填）】即可发布，无需负责拉群、暖场。3.3 朋友动态 (Moments)设计风格：类朋友圈的瀑布流/列表设计，注重真实感和生活气息。功能描述：活动返图与日常分享：用户可以发布图文动态，分享生活点滴。活动强关联 (Activity Ref)：如果动态是关于某次通过 APP 发起的活动，动态下方会带有专属标签（如：🔗 参与了：周末郊区露营）。点击标签可查看该类型活动，形成闭环种草。轻互动：支持点赞（Heart）和评论（Comment）。3.4 朋友通讯录 (Contacts)设计风格：干净清晰的列表设计，带顶部搜索栏。功能描述：好友列表：展示好友的圆角头像、昵称和个性签名/备注。系统级联络：每个好友右侧提供“短信”和“电话”快捷按钮。点击后直接唤起手机原生的短信和拨号界面，不在 APP 内部留存聊天记录，保护隐私且轻量。3.5 我的主页与设置 (Profile & Settings)设计风格：顶部个性化资料卡，下方为卡片化选项列表；二级页面采用 iOS 原生级别的右侧滑入动画。功能描述：个人资料：展示头像、昵称、社交标签。社交数据：直观展示用户的社交活跃度（发起的局、参与的局、我的朋友数量）。行程管理：提供“即将开始的行程”和“我看过的聚会”入口。设置与隐私 (沉浸式二级页)：包含账号安全、青少年模式、新消息通知等常规设置。特色功能：隐身模式（开启后不被推荐给可能认识的朋友，保护社交边界感）。4. UI/UX 视觉规范主色调：Indigo（靛蓝色，#4F46E5），代表现代、活力与可靠。头像规范：全站统一采用圆角矩形 (Squircle)。若用户未上传头像，系统根据其名字首字母和哈希算法，自动生成固定底色的纯文本头像。排版细节：大量使用毛玻璃效果（Backdrop Blur）、柔和阴影和渐变遮罩，提升卡片的文字可读性和整体的“高级感”。交互动画：页面切换（如进入设置页）需符合 iOS 原生滑动反馈，卡片滑动带有惯性阻尼（Snap-mandatory）效果。5. 商业化预想 (可选)线下商家合作 (B端)：剧本杀店、密室、餐厅、滑雪俱乐部可以入驻成为“认证局长”，发布带有商家专属标签的活动卡片。增值服务 (C端)：超级曝光卡（让自己的局被更多人看到）、特殊颜色的卡片背景、更多的动态配图数量等。6. 技术路线建议 (MVP版本)前端：React Native (Expo) 或 Flutter，一套代码同时编译 iOS 和 Android 版本。后端：Node.js / Go，配合 PostgreSQL 或 MongoDB。UI 组件库：推荐使用 Tailwind CSS 结合原生组件手写，以保证极致的定制化视觉效果。文档版本：v1.0生成日期：2026-05-10
